## 四、执行策略

### 4.1 团队分工
- 20个agent按工作包1:1分配
- 每个agent独立负责一个工作包的完整交付
- 香草（PM）负责验收、集成、git提交
- 所有工作包无代码依赖，可完全并行

### 4.2 代码规范
- 严格复用现有模式: Definition + Handler + Tools数组
- 工具命名: `wps_{app}_{action}`
- 文件大小: 每文件不超过600行
- 调用方式: `wpsClient.executeMethod(action, params, appType)`
- 类型: RegisteredTool = { definition: ToolDefinition, handler: ToolHandler }

### 4.3 验证方案
1. **静态验证**: tsc --noEmit 编译通过
2. **单元测试**: 工具注册和handler mock测试
3. **集成验证**: allTools.length 和工具名唯一性检查
4. **端到端**: MCP Server tools/list 返回全量工具

## 六、风险与对策

1. **index.ts合并冲突**: 每阶段结束后由PM统一合并
2. **现有工具重复**: agent扩展时已覆盖部分P1工具，需去重
3. **文件过大**: 严格控制每文件600行上限，超出则拆分
4. **wps_execute_method兜底**: 即使专业工具未覆盖，万能分发器可间接调用全部224个action
